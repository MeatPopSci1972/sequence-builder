## What this project is

Single-file, zero-dependency static HTML application for building UML sequence diagrams visually.
Outputs PlantUML or Mermaid syntax. Runs entirely in the browser.
Deployment: git push → GitHub Pages serves sequence-builder.html directly.

## Architecture in one diagram

```
User event
    |
    v
store.dispatch(action)          <- ONLY mutation entry point
    |
    |-- log.push(action)
    |-- pushSnapshot()           <- if meta.undoable !== false
    |-- handler mutates state    <- try/catch: pops log on throw
    +-- emit(event, payload)
              |
              v
        store.on() listeners     <- registered in HTML, never in store
              |
              |-- update uiState
              +-- render() or requestRender()
                      |
                      v
                  SVG layers (inside <g id="viewport">)
                  lifelines -> fragments -> messages -> notes -> actors
```

**Invariant:** store never reads uiState. store never calls render().

## State model

```js
// store.state — serializable, persistable, narrow
{
  actors:    [{ id, x, label, type, emoji? }],
  messages:  [{ id, fromId, toId, label, kind, direction, y,
                protocol?, port?, auth?, dataClass? }],
  notes:     [{ id, x, y, text }],
  fragments: [{ id, x, y, w, h, kind, cond }],
  nextId:    Number
}

// uiState — never persisted, resets on reload
{
  selected:       null | { ...obj, _type, _ref, _preview? },
  mode:           'select' | 'connect' | 'place-actor',
  connectFrom:    null | actorId,
  pendingActorId: null | actorId,
  format:         'plantuml' | 'mermaid',
  placeGhostX:    null | Number
}
```

## The six things most likely to confuse a fresh AI instance

**1. Never edit @@STORE-START...@@STORE-END in the HTML.**
Generated by build.js from sequence-builder.store.js.

**2. _wrapSelected freezes a COPY, not the live store object.**
Object.freeze({ ...obj }) — the original stays mutable.

**3. render() is not requestRender().**
render() is immediate — after mutations. requestRender() is rAF-batched — during drag.

**4. const state = _store.state is a live alias, not a copy.**
Writing directly bypasses log and undo.

**5. REFLOW_ACTORS for batch actor moves.**
One undoable step for multiple actor positions.

**6. _parseUML returns { actors, messages, warnings }.**
warnings is always present. Import blocked when any warning has non-null hint.
