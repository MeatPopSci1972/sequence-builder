## What this project is

Single-file, zero-dependency static HTML application for building UML sequence diagrams visually. Outputs PlantUML or Mermaid syntax. Runs entirely in the browser — no server, no npm, no build pipeline except the one-step `node build.js` that syncs the store module into the HTML.

**Deployment:** `git push` → GitHub Pages serves `sequence-builder.html` directly. Done.

**v0.9.18 — Arrow regex and <-> fix**
ARROW_RE was using \S+ for actor capture which grabbed Alice->>Bob as one token in no-space Mermaid style. Fixed to [^<>\-\s]+ which stops at arrow characters. Added <-> (bidirectional, direction:'both') and <--> (dashed bidirectional, kind:'async', direction:'both') to the arrow token set. Labelless arrows (A -> B with no : label) now parse correctly — label group is optional. Regression test added. 41 tests total.

**v0.9.19 — Strict parser with per-line warnings**
_parseUML now collects warnings instead of silently dropping unrecognised lines. Two warning classes:
- Hint warnings (w.hint !== null): fixable user mistakes (e.g. actor Alice a without as). Import blocked until resolved. Hint shown inline.
- Informational warnings (w.hint === null): unsupported-but-harmless directives. Import still allowed. Lines surfaced in amber so user knows what was ignored.

Modal preview shows: green count on clean input, amber per-line list with line numbers and hints on fixable errors, red message on unrecognisable format or no actors. Parser returns { actors, messages, warnings } — consumers must handle the warnings field. 43 tests total.

## Architecture in one diagram

```
User event
    |
    v
store.dispatch(action)          <- ONLY mutation entry point
    |
    |-- log.push(action)
    |-- pushSnapshot()           <- if meta.undoable !== false
    |-- handler mutates state    <- try/catch: pops log on throw
    +-- emit(event, payload)
              |
              v
        store.on() listeners     <- registered in HTML, never in store
              |
              |-- update uiState
              +-- render() or requestRender()
                      |
                      |-- render()        - mutations (immediate)
                      +-- requestRender() - drag (rAF-batched)
                              |
                              v
                          SVG layers (inside <g id="viewport">)
                          lifelines -> fragments -> messages -> notes -> actors
```

**Invariant:** store never reads uiState. store never calls render().

## State model

```js
// store.state — serializable, persistable, narrow
{
  actors:    [{ id, x, label, type, emoji? }],
  messages:  [{ id, fromId, toId, label, kind, direction, y,
                protocol?, port?, auth?, dataClass? }],
  notes:     [{ id, x, y, text }],
  fragments: [{ id, x, y, w, h, kind, cond }],
  nextId:    Number
}

// uiState — never persisted, resets on reload
{
  selected:       null | { ...obj, _type, _ref, _preview? },
  mode:           'select' | 'connect' | 'place-actor',
  connectFrom:    null | actorId,
  pendingActorId: null | actorId,
  format:         'plantuml' | 'mermaid',
  placeGhostX:    null | Number   // transient — place-actor mode only
}
```

**Persistence rule:** if it survives a browser refresh -> store.state. If it resets on reload -> uiState.

## The six things most likely to confuse a fresh AI instance

**1. Never edit the @@STORE-START...@@STORE-END block in the HTML.**
It's generated. Edit sequence-builder.store.js, run tests, run node build.js.

**2. _wrapSelected freezes a COPY, not the live store object.**
Object.freeze({ ...obj }) — the original stays mutable for dispatch. Freezing obj directly silently blocks all future mutations on that object in non-strict script context.

**3. render() is not requestRender().**
render() is immediate — use after mutations. requestRender() is rAF-batched — use during drag. Never swap them.

**4. const state = _store.state is a live alias, not a copy.**
Reading it is fine everywhere. Writing to it directly (state.actors.push(...)) bypasses the log and undo won't see it.

**5. REFLOW_ACTORS exists for batch actor moves.**
Dispatching N individual MOVE_ACTOR actions pushes N snapshots. Use REFLOW_ACTORS with payload.positions: [{id, x}] when multiple actors need to move in one undoable step.

**6. _parseUML returns { actors, messages, warnings } — not just { actors, messages }.**
The warnings array is always present (may be empty). _umlImportPreview blocks import when any warning has a non-null hint. Unknown/unsupported lines produce { hint: null } warnings — informational only, import still allowed. Do not remove the warnings field from the return value.

## Key settled decisions (do not relitigate without updating the main handoff)

- Narrow store: store.state is the diagram record only. uiState is separate.
- Synchronous dispatch: no async in the dispatch path, ever.
- REFLOW_ACTORS for batch moves: one snapshot covers all actor positions.
- Click on message = select only: kind changes via Properties panel. dblclick cycles direction. Final (Option A, v0.9.13).
- save fires on every render(): save-on-render, not save-on-dispatch. Acceptable at current diagram sizes.
- Fragment layer order: lifelines first, then fragments, then everything else on top.
- _ref is a frozen copy: _wrapSelected freezes { ...obj }, not obj itself.
- UML import is strict, not permissive: parser blocks on fixable mistakes (hint warnings) rather than silently importing a broken diagram. Do not revert to silent-drop behaviour.
- Import button disabled until parse is clean of hint warnings: _umlImportPreview gates on warnings.some(w => w.hint !== null). Informational skips (null hint) do not block.

## UML import — supported syntax reference

```
PlantUML:
  @startuml / @enduml        optional wrapper
  participant Label           declares an actor
  participant "Long Label" as A   declares with alias
  actor Label as A            same as participant
  A -> B : label              sync, rightward
  A <- B : label              sync, leftward (flipped to from=B, to=A)
  A <-> B : label             bidirectional, direction=both, kind=sync
  A --> B : label             async, rightward
  A <-- B : label             async, leftward
  A <--> B : label            bidirectional, direction=both, kind=async
  A -->> B : label            return
  A ->> B : label             async (Mermaid-style also accepted)
  ' comment                   stripped
  // comment                  stripped
  label is optional           A -> B  (no colon) is valid

Mermaid:
  sequenceDiagram             required header
  participant Label           declares an actor
  participant Label as A      declares with alias
  A->>B: label                async (no spaces required around arrow)
  A-->>B: label               return
  A->B: label                 sync
  A-->B: label                async

NOT supported (silently skipped, informational warning):
  title, note, loop, alt, else, opt, group, box, skinparam,
  autonumber, activate, deactivate, return

INVALID (hint warning shown, import blocked):
  actor Label Alias            missing 'as' keyword
```
