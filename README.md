# SequenceForge

A browser-based visual UML sequence diagram builder, built as a prototype-to-production teaching project.

Draw actors, connect them with typed messages, export to PlantUML or Mermaid syntax — or paste existing UML text to import a diagram. Zero dependencies, zero build step, single HTML file.

## What it does

- **Visual sequence diagram editor** — drag actors, connect with messages, reorder by dragging between positions
- **PlantUML + Mermaid export** — switch formats live; output updates on every change
- **UML text import** — paste PlantUML or Mermaid syntax; parser validates with per-line warnings before committing
- **Message types** — sync, async, return; direction cycles on double-click; kind set via Properties panel
- **Fragments** — alt, loop, opt, par, ref boxes with SE-corner resize handle
- **Notes** — freeform text annotations, draggable
- **Place-actor mode** — click canvas to position a new actor at an exact X coordinate
- **Actor drag reflow** — drop an actor between two others; all actors reflow to clean even spacing in one undoable step
- **Full undo/redo** — 50-step stack, branching history (new mutation after undo clears redo)
- **localStorage persistence** — diagram survives browser refresh; JSON export/import for sharing
- **JSON import/export** — save and restore full diagram state as a `.json` file
- **Debug console** — built-in action log and verbose toggle for development

## Prototype-to-production teaching notes

This project deliberately preserves the full iterative history — bugs found mid-session, decisions made and reversed, architectural patterns chosen for a zero-dependency constraint. The intent is to show:

1. **Single-file deployment is a valid constraint** — GitHub Pages serves one HTML file; every feature ships by pushing one file
2. **Narrow store as the only mutation path** — `store.dispatch()` is the single entry point; `uiState` is separate and never persisted
3. **Validate before optimize** — rAF batching for drag was added only after the render path was provably correct; targeted SVG redraws are next on the same principle
4. **Tests as a contract, not coverage** — the 43-test suite targets store behaviour, not render output; it runs in Node with zero browser dependencies
5. **Strict over permissive** — the UML parser blocks on fixable mistakes with hints rather than silently importing a broken diagram

## Running it

Open `sequence-builder.html` directly in any modern browser. No build step, no server, no dependencies.

To develop:

```powershell
# After editing sequence-builder.store.js:
node build.js                      # sync store into HTML
node sequence-builder.test.js      # run 85 tests
# Expected: 85 passed | 0 failed | 85 total
```

Full gate (build + syntax check + tests):

```powershell
node build.js && `
sed -n '/<script>/,/<\/script>/p' sequence-builder.html `
  | sed '1s/<script>//' | sed '$s/<\/script>//' > sf-script.js && `
node --check sf-script.js && `
node sequence-builder.test.js
# Expected: 85 passed | 0 failed | 85 total
```

## UML import — supported syntax

```
PlantUML:
  @startuml / @enduml            optional wrapper
  participant Label               declares an actor
  participant "Long Label" as A   declares with alias
  actor Label as A                same as participant
  A -> B : label                  sync
  A <- B : label                  sync leftward
  A <-> B : label                 bidirectional (direction=both)
  A --> B : label                 async
  A <--> B : label                async bidirectional
  A -->> B : label                return
  label is optional               A -> B  is valid

Mermaid:
  sequenceDiagram                 required header
  participant Label as A          declares with alias
  A->>B: label                    async (no spaces required)
  A-->>B: label                   return
```
